# Melony AI

**Melony AI** - это высокопроизводительный асинхронный сервис для автоматизированного скрининга и валидации сотен резюме на соответствие кастомным критериям отбора за считанные минуты. 

В основе системы лежит отказоустойчивый оркестратор, координирующий работу двух независимых LLM по принципу судейства и перекрестной проверки.

---

##  Технологический стек

* **Backend:** FastAPI (Python 3.12)
* **Task Queue & Broker:** Celery + Redis
* **Database:** PostgreSQL
* **AI Engine:** LLM API Orchestration / Deepseek-v4-flash

---

##  Архитектура и пайплайн обработки

Процесс анализа резюме полностью асинхронен и изолирован от основного потока запросов:

1. **Прием запроса (`FastAPI`)** Клиент отправляет резюме на сервер. FastAPI мгновенно извлекает требования к вакансии из PostgreSQL, формирует задачу и передает её в брокер сообщений (Redis). Клиент сразу получает ID задачи со статусом `в обработке` (HTTP 202).

2. **Очередь задач (`Celery`)** Воркеры Celery подхватывают задачу из Redis и изолированно запускают процесс фонового анализа, разгружая основной веб-сервер.

3. **Многоагентный анализ (`LLM Orchestrator`)** Две независимые языковые модели тщательно проверяют текст резюме. Первая модель формирует результат, вторая - валидирует и перепроверяет выводы первой, исключая галлюцинации и ошибки.

4. **Доставка результата (`Client Delivery`)** Финальный структурированный ответ отдается клиенту в удобном для интеграции формате.

##  Оптимизация и отказоустойчивость

Для обеспечения высокой пропускной способности и стабильности системы под нагрузкой реализованы следующие механизмы:

* **Идемпотентность и защита от дублей:** Хеширование данных кандидатов предотвращает повторную отправку и платную обработку одного и того же резюме несколько раз.
* **Кэширование данных:** Хеширование ключей и требований клиентов в оперативной памяти минимизирует задержки и снижает нагрузку на БД.
* **State Management:** Логирование и фиксация промежуточных состояний тасок позволяют системе быстро восстанавливаться после непредвиденных сбоев или перезагрузки воркеров без потери прогресса.

##  Как использовать

Система поддерживает два формата взаимодействия:
1. **Web UI:** Использование готового веб-интерфейса для ручной загрузки.
2. **API Client:** Интеграция в существующие HR-системы компании через REST API для полной автоматизации потока резюме.
